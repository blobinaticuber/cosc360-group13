# COSC 360 Project Proposal

## Project Description

Virtual Library and Book Exchange Platform

The project is to create a platform where users can share, borrow, and exchange books with others in their local community. Users will be able to record which books they have and are willing to lend out, and can also search from books available in their area and request to borrow them. Users can filter books by availability, genre, author(s), and title.

## Requirements List

Website user’s objectives:
- Browse the site without registering.
- Register at the site by providing their name, e-mail and image (optional).
- Log in by providing a username (or email) and password.Search for listed books by keywords (author’s name, genre, etc.) without registering.
- View and edit their profile.
- Create listings and comments when logged into the site.
Request a listed book.

Website administrator’s objectives:
- Search for users by name, email or post.
- Enable/disable user accounts.
- Edit/remove posted listings.

Additional Requirements:
- Hand-styled layout with contextual menus. (i.e., when a user is logged in, some menus may look different to reflect that.)
- 2 or 3 column layout using appropriate design principles (i.e. highlighting nav links when hovered over, etc).
- Form validation will be carried out on the frontend.
- Business logic will be implemented on the back-end, which will expose a Restful API written with ExpressJS.
- Requests to the back-end API will be made with the Fetch API.
- Persistent data will be stored in a MongoDB database.
- Appropriate security will be used for sensitive data.
- The front-end site maintains state and reflects it in the UI.
- The site adheres to responsive design philosophy.
- User images and profiles can be stored in the database.
- Simple book grouping and displays.
- Breadcrumb navigation will be used where appropriate.
- The client and server should each be able to gracefully handle and recover from errors.
